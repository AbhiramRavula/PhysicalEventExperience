1. Executive Summary
StadiumIQ is a unified, mobile-first platform designed to transform the attendee experience at large-scale sporting venues — football/soccer stadiums, cricket grounds, and multi-sport arenas — supporting up to 80,000 concurrent attendees per event. The platform combines smart wayfinding, real-time crowd intelligence, virtual queue management, food and merchandise pre-ordering, live event notifications, and an operator command dashboard into one coherent ecosystem.

🎯  Engineering Mandate
Build a fault-tolerant, horizontally scalable platform that delivers sub-second wayfinding updates, <30s emergency alert propagation, and in-seat F&B ordering with 99.9% uptime during live events — across iOS, Android, Web PWA, and In-venue Kiosks.

🚧  Problem Statement
Large-scale events suffer from congestion at gates, concessions, restrooms, and parking. Bottlenecks degrade fan experience, reduce revenue, and create safety risks. Operators lack real-time crowd visibility and dynamic communication tools.

5.3 Communication Patterns
REST over HTTPS — all request/response client-to-backend calls.
WebSocket (Socket.io / native WS) — live heatmap, operator dashboard, order status updates.
Kafka topics — internal async bus between microservices (crowd telemetry, order events, alert triggers).
Server-Sent Events (SSE) — fallback for clients that cannot maintain WebSocket connections.
FCM / APNs — push notifications for mobile; Web Push for PWA browsers.


6. Core Data Models
6.1 Venue & Zone Schema
CREATE TABLE venues (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name         VARCHAR(200) NOT NULL,
  city         VARCHAR(100),
  country_code CHAR(2),
  capacity     INTEGER,
  geom         GEOMETRY(Polygon, 4326),
  created_at   TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE zones (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  venue_id     UUID REFERENCES venues(id),
  name         VARCHAR(100) NOT NULL,
  type         VARCHAR(50),   -- gate|concourse|stand|restroom|concession|exit
  floor        SMALLINT DEFAULT 0,
  capacity     INTEGER,
  geom         GEOMETRY(Polygon, 4326),
  centroid     GEOMETRY(Point, 4326)
);

CREATE TABLE beacons (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  zone_id      UUID REFERENCES zones(id),
  hardware_id  VARCHAR(64) UNIQUE NOT NULL,
  location     GEOMETRY(Point, 4326),
  floor        SMALLINT,
  battery_pct  SMALLINT,
  last_seen    TIMESTAMPTZ
);

6.2 Crowd Telemetry (TimescaleDB Hypertable)
CREATE TABLE crowd_telemetry (
  time         TIMESTAMPTZ NOT NULL,
  zone_id      UUID NOT NULL,
  device_count INTEGER,       -- aggregate only; no individual IDs
  density_pct  NUMERIC(5,2),  -- 0.00 – 100.00
  flow_dir     VARCHAR(16),   -- inbound|outbound|static
  source       VARCHAR(20)    -- beacon|sensor|cv_camera|wifi
);
SELECT create_hypertable('crowd_telemetry','time');
CREATE INDEX ON crowd_telemetry (zone_id, time DESC);

6.3 Orders Schema
CREATE TABLE orders (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  attendee_id     UUID NOT NULL,
  event_id        UUID NOT NULL,
  venue_id        UUID REFERENCES venues(id),
  status          VARCHAR(20) DEFAULT 'pending',
                  -- pending|confirmed|preparing|ready|collected|cancelled
  order_type      VARCHAR(20),  -- food|beverage|merchandise
  pickup_zone_id  UUID REFERENCES zones(id),
  total_amount    NUMERIC(10,2),
  currency        CHAR(3) DEFAULT 'INR',
  payment_ref     VARCHAR(100),
  created_at      TIMESTAMPTZ DEFAULT now(),
  ready_at        TIMESTAMPTZ,
  collected_at    TIMESTAMPTZ
);

CREATE TABLE order_items (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id    UUID REFERENCES orders(id) ON DELETE CASCADE,
  product_id  UUID NOT NULL,
  quantity    SMALLINT NOT NULL,
  unit_price  NUMERIC(10,2)
);

8.2 Cloud Infrastructure (AWS Reference)
Compute: EKS (Kubernetes 1.29+); node groups auto-scaled on CPU + custom Kafka lag metric.
Database: RDS Aurora PostgreSQL 15 (multi-AZ) + TimescaleDB on EC2; ElastiCache Redis for hot-path caching.
Messaging: MSK (Managed Kafka) — topics: crowd.telemetry, order.events, alert.commands, beacon.heartbeat.
Storage: S3 for venue map tiles + media; CloudFront CDN for tile distribution.
Edge: Wavelength Zones or on-premise k3s nodes for <10ms intra-venue latency.
Observability: OpenTelemetry → Grafana stack (Tempo, Loki, Prometheus); PagerDuty alerting.

15.3 k6 Load Test Script
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Rate } from 'k6/metrics';
const errorRate = new Rate('errors');

export const options = {
  stages: [
    { duration: '2m',  target: 1000 },  // ramp up
    { duration: '5m',  target: 5000 },  // sustained peak (gate rush)
    { duration: '10m', target: 5000 },  // soak
    { duration: '2m',  target: 0    },  // ramp down
  ],
  thresholds: {
    http_req_duration: ['p(99)<800'],   // wayfinding SLA
    errors:            ['rate<0.01'],   // <1% error rate
  },
};

export default function () {
  const login = http.post(`${__ENV.BASE_URL}/auth/login`,
    JSON.stringify({ barcode: `TICKET-${Math.random()}`, device_id: `dev-${__VU}` }),
    { headers: { 'Content-Type': 'application/json' } }
  );
  check(login, { 'login 200': r => r.status === 200 });
  const token = login.json('access_token');

  const route = http.post(`${__ENV.BASE_URL}/venues/venue-001/route`,
    JSON.stringify({ from_zone: 'zone-gate-a', to_zone: `zone-stand-${Math.ceil(Math.random()*2)}` }),
    { headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' } }
  );
  check(route, { 'route 200': r => r.status === 200 });
  errorRate.add(route.status !== 200);
  sleep(1);
}

function getDensity(pattern, min) {
  const r = () => Math.random() * 0.1;
  switch(pattern) {
    case 'entry_surge':   return min < 30  ? 0.4 + r() + 0.4 : 0.1 + r();
    case 'entry_light':   return min < 20  ? 0.3 + r()       : 0.05;
    case 'halftime_peak': return (min>44&&min<60) ? 0.7+r()+0.2 : 0.15+r();
    case 'stable_high':   return 0.82 + r();
    case 'exit_surge':    return min > 88  ? 0.6 + r() + 0.3 : 0.05;
    default:              return 0.2 + r() * 3;
  }
}

async function run() {
  await producer.connect();
  let minute = 0;
  setInterval(async () => {
    minute = (minute + 1) % 95;
    for (const zone of ZONES) {
      const density = Math.min(1, getDensity(zone.pattern, minute));
      await producer.send({ topic: 'crowd.telemetry', messages: [{ value: JSON.stringify({
        time: new Date().toISOString(),
        zone_id: zone.id,
        device_count: Math.round(zone.capacity * density),
        density_pct: +(density * 100).toFixed(2),
        flow_dir: density > 0.5 ? 'inbound' : 'static',
        source: 'ble_simulator'
      })}]});
    }
    console.log(`[BLE-SIM] minute=${minute} — 12 zone events emitted`);
  }, 10000);  // every 10s = 10s heatmap cadence
}
run().catch(console.error);

19. Open Engineering Questions
❓  Indoor Positioning SDK Choice
Evaluate: (a) native BLE trilateration custom, (b) Mapbox Indoor SDK, (c) IndoorAtlas, (d) Pointr. Decision needed before Sprint 1 kickoff to avoid beacon abstraction layer rework.

❓  Offline Grace Period
If attendee loses connectivity mid-navigation — options: (a) fixed 5-min cache TTL, (b) invalidate on next BLE heartbeat mismatch, (c) keep until explicit re-sync. Must be decided before Sprint 1 coding freeze.

❓  Kiosk Update Strategy
(a) OTA via Service Worker nightly, (b) operator triggers update from dashboard, (c) forced refresh at fixed time. Impacts deployment pipeline and rollback strategy.

❓  Multi-tenancy Model
Hard isolation (per-venue Kubernetes namespace) vs shared cluster with row-level security. Decision affects onboarding automation, cost, and compliance boundary.

❓  Data Ownership & Licensing
Who owns aggregate crowd analytics — venue, organiser, or StadiumIQ? Licensing model required before pilot contracts can be signed.
